# Button

Buttons trigger actions in the UI. Prefer one primary button per view, and use
secondary styles for supporting actions.

## Usage guidelines

- Use `default` for the primary action.
- Use `outline` or `secondary` for supporting actions.
- Reserve `destructive` for irreversible actions.
- Use `asChild` with an anchor for navigation links.
